# Discord 브리핑 메시지 형식 가이드

## 메시지 구조

```
━━━━━━━━━━━━━━━━━━━━━━━
  브리핑 · MM/DD 요일요일
━━━━━━━━━━━━━━━━━━━━━━━

💼 업무  (HH:MM~HH:MM)
  🗓 10:00  팀 스탠드업
  ⬜ 전략 자료 준비  ★
  ⬜ 월간 보고서 초안

🌙 개인  (19:00~)
  🗓 19:00  심리상담
  ⬜ 운동 30분

📦 백로그 → Todoist
  개발 환경 업데이트 · 문서 정리

━━━━━━━━━━━━━━━━━━━━━━━
  오늘도 하나씩 차분하게. 💪
━━━━━━━━━━━━━━━━━━━━━━━
```

## 분류 로직

### GCal 이벤트 분류
1. `colorId == "4"` → personal
2. 제목에 개인 키워드 포함 (`심리상담`, `병원`, `운동`, `자세교정`, `가족`, `약속`) → personal
3. 제목에 업무 키워드 포함 (`미팅`, `스탠드업`, `리뷰`, `발표`, `보고`) → work
4. 시작 시간 09:00~17:59 → work, 그 외 → personal

### Todoist 태스크 분류
| root_project_name | 분류 |
|-------------------|------|
| 業務リスト | work |
| 자기계발 | personal |
| 간단일 리스트 | backlog |
| 기타 / 매핑 불명 | personal (fallback) |

## 섹션별 처리

- **💼 업무**: 평일에만 표시. 업무 이벤트 + 업무 태스크. 업무 이벤트 시간대로 헤더 표시.
- **🌙 개인**: 개인 이벤트 + 개인 태스크. 없으면 섹션 전체 생략.
- **📦 백로그**: 백로그 태스크를 한 줄에 ` · ` 구분자로 나열. 없으면 생략.
- **하단 코멘트**: LLM이 생성한 한 줄 코멘트 (30자 이내, 이모지 1개 포함).

## 아이템 표기

- GCal 이벤트: `🗓 HH:MM  제목`
- Todoist 태스크: `⬜ 내용` (p1이면 `  ★` 접미어)
- 각 블록 내 이벤트는 시작 시간 오름차순 정렬

## LLM 코멘트 규칙

- 한 줄, 30자 이내
- 한국어, 친근하고 실용적
- 이모지 1개 포함
- 매번 다른 표현
- 오늘 요일, 일정 수, p1 태스크 수 컨텍스트 활용
